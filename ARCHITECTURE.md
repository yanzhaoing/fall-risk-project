## 三、核心设计理念

### ❌ 错误方向（之前的做法）
- 二分类（Fall / ADL）
- 跌倒动作识别（事后检测）
- 单模态（仅骨架）
- 全局模型（无个性化）

### ✅ 正确方向
- **连续风险评分（0-100）**
- **跌倒风险前置预判**（在跌倒发生前预警）
- **多模态融合**（行为 + 环境 + 生理）
- **个性化基线**（每位老人独立正常状态）
- **分级预警 + 联动响应**

## 五、模块拆解与状态

### Module 1：姿态估计
- **输入：** 视频帧 / 摄像头取流
- **输出：** COCO 17 关键点序列
- **技术：** MediaPipe Pose → COCO 映射
- **状态：** ⏳ 待实现（需要萤石摄像头或 UP-Fall 数据）

### Module 2：步态特征提取
- **输入：** 骨架序列（T × 17 × 2）
- **输出：** 步态特征向量
- **特征：**
  - 步速（hip 中心移动速度）
  - 步幅（双脚距离变化）
  - 身体重心摆动（hip 中心标准差）
  - 关节角度变化率（膝关节、髋关节角度的导数）
  - 平衡指标（重心投影在支撑面内的比例）
  - 步态周期稳定性（步态周期的标准差）
- **状态：** ⏳ 待实现

### Module 3：风险评分模型
- **输入：** 步态特征 + 多模态特征 + 个性化偏离度
- **输出：** 0-100 连续风险分数
- **架构：** LSTM/Transformer 编码器 + MLP 评分头
- **损失：** MSE Loss + Pairwise Ranking Loss
- **评估：** MAE、Spearman 相关系数、风险等级准确率
- **状态：** ✅ 基础 pipeline 已跑通（GaitRiskScorer, --task regression）

### Module 4：个性化基线
- **输入：** 某位老人的历史正常数据
- **输出：** 该老人的正常状态特征分布（均值、方差）
- **逻辑：** 新数据与基线的偏离度 → 额外风险加成
- **实现：** PersonalizedBaseline + BaselineManager
  - 校准期：采集正常数据，计算均值/标准差
  - 实时期：z-score 偏离度（每个特征几个标准差）
  - 更新期：指数移动平均（alpha=0.05），适应缓慢变化
  - 持久化：JSON 文件保存/加载
  - 多用户管理：BaselineManager 统一管理
- **状态：** ✅ 已实现并测试通过

### Module 5：多模态融合
- **输入：** 行为特征 + 环境特征 + 生理特征
- **输出：** 融合特征向量
- **技术：** Attention-based Fusion / Cross-Modal Transformer
- **子模块：**
  - 行为模态：步态特征（16维）✅
  - 环境模态：环境风险特征（12维）✅
  - 轨迹模态：轨迹分析特征（6维）✅
  - 生理模态：暂用步态间接代理
- **融合模型：** MultiModalRiskScorer（GaitLSTM + GatedFusion + RiskHead）✅
- **状态：** ✅ 基础 pipeline 已跑通（--task multimodal）

### Module 6：分级预警
- **输入：** 风险分数
- **输出：** 预警等级 + 响应动作
- **逻辑：** 阈值分级（0-30 / 31-60 / 61-80 / 81-100）
- **实现：** AlertSystem 类
  - 时间平滑：指数加权平均，避免分数波动导致频繁切换
  - 持续时间：高风险持续 >30s 才通知，极高风险 >5s
  - 冷却期：通知后 5 分钟内不重复通知
  - 响应策略：4 级 × 5 维（摄像头/录像/通知/音箱/频率）
- **状态：** ✅ 已实现并测试通过

### Module 7：萤石集成
- **功能：** 摄像头取流、事件推送、设备联动
- **依赖：** 萤石开放平台 API（报名后获取权限）
- **状态：** ⏳ 等待报名（5.30）

## 六、数据策略

### 主训练集：UP-Fall
- 17 人，30fps，真实跌倒 + 日常活动
- 需要下载（~15GB）
- 更接近实际场景

### 辅助数据：NTU RGB+D
- 已转换（16,995 COCO JSON）
- 可用于预训练姿态估计 / 步态特征提取
- ⚠️ 不能直接当"风险数据"用（是动作识别数据）

### 风险标注策略
- UP-Fall 中的 ADL → 低风险（0-30）
- UP-Fall 中的接近跌倒动作 → 中风险（31-60）
- UP-Fall 中的跌倒前几帧 → 高风险（61-80）
- 实际跌倒 → 极高风险（81-100）
- 根据动作类型 + 时序位置映射成 0-100 连续标注
