# 评审证据矩阵

## 目标

把“系统做了什么”和“能拿什么证明”一一对应，避免初审时被认为只会做概念展示。

| 评审关注点 | 当前证据 | 还缺什么 |
| --- | --- | --- |
| 前置预判能力 | 风险评分模型、演示场景分级 | 真实居家样例 |
| 过程识别能力 | 视频分析、场景连续输出 | 真实视频运行记录 |
| 分级预警闭环 | AlertSystem、演示页面响应动作 | 实际通知或联动截图 |
| 多模态思路 | 步态、环境、基线偏移三路说明 | 平台或设备联动证据 |
| 技术有效性 | 公开数据集离线评测、功能测试报告 | 最终统一实验表 |
| 平台要求 | 萤石 API 客户端代码 | 平台调用截图/日志 |
| 落地价值 | 适老化场景说明、居家风险闭环设计 | 真实部署或模拟部署照片 |

## 使用方法

后续每补一项材料，就把它挂到对应证据栏里。这样写专项研究报告和答辩稿时，不会出现“讲了很多，但没有证据支撑”的问题。
