# 参赛重做说明

本轮重做的目标不是把仓库包装成“已经完全落地的真实产品”，而是把它提升为：

- 技术路线清晰
- 系统闭环完整
- 指标口径明确
- 评审可理解、可验证、可演示

## 当前主线

1. 公开骨架数据集离线验证
2. 步态特征风险评分模型
3. 四级风险分级预警
4. 演示页面呈现系统闭环
5. 后续补齐真实居家场景与萤石平台证据

## 已修正的问题

1. 风险代理标签优先使用 `action_id`，避免目录名异常导致标签映射失真。
2. 时序模型训练后，测试集评估会重新加载 `best_model.pt`，避免拿最后一轮权重冒充最佳模型。
3. 比赛演示服务缺失前端资源的问题已补齐。

## 后续重点

1. 补充真实测试视频与平台接入证据
2. 统一报告、演示和结果文件中的指标口径
3. 增加实测截图、流程图和部署说明
